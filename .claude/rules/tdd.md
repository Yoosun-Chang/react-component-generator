# TDD Rules

> **경고: 이 규칙은 Rigid — 상황에 맞게 변형하지 마라.**
> 프로젝트의 `AGENTS.md`에 TDD 관련 지침이 있으면 그쪽이 우선한다. 이 파일은 fallback 기본값이다.

---

## 1. 적용 기준

### TDD를 **반드시** 적용해야 하는 대상

| 대상 | 예시 |
|------|------|
| 비즈니스 로직 | AI 응답 후처리, 컴포넌트 코드 검증/정제 함수 |
| API 엔드포인트 | `/api/generate`, `/api/config` 핸들러 |
| 유틸리티 함수 | `resolveApiKey`, `sanitizeCode`, 파싱 함수 |
| 훅(Hook) 로직 | `useComponentGenerator` 내부 상태/분기 |
| 버그 수정 | 수정 전 버그를 재현하는 실패 테스트 먼저 작성 |

### TDD가 **불필요**한 대상

| 대상 | 이유 |
|------|------|
| 타입 정의 (`.d.ts`, `type`, `interface`) | 컴파일러가 검증 |
| 설정 파일 (`vite.config.ts`, `eslint.config.js` 등) | 런타임 통합으로 확인 |
| 순수 UI 렌더링 (JSX 마크업) | 시각적 검증이 주목적 |
| SQL / 스키마 마이그레이션 | DB 통합 테스트 범주 |

## 2. RED-GREEN-REFACTOR 사이클

### RED — 실패하는 테스트 작성

- **하나의 동작 = 하나의 테스트.** 여러 동작을 한 테스트에 묶지 마라.
- 테스트를 **반드시 실행**해서 실패를 확인한다. 실행 없이 GREEN으로 넘어가지 마라.
- 실패 이유가 **"기능 미구현"** 이어야 한다. `import` 오류나 타입 오류로 실패하면 테스트 설정 문제다 — 먼저 고쳐라.
- 테스트 이름은 동작을 서술한다: `"should return false when API key is missing"`.

### GREEN — 테스트를 통과시키는 최소 코드 작성

- **최소한의 코드만** 작성한다. 테스트가 요구하지 않는 기능은 추가하지 마라 (YAGNI).
- 신규 테스트뿐 아니라 **기존 테스트 전체**가 통과하는지 확인한다.
- 코드가 지저분해도 괜찮다 — GREEN이 목표다. 정리는 REFACTOR에서.

### REFACTOR — 동작 유지하며 코드 정리

- **중복 제거, 이름 개선, 헬퍼 추출**만 허용한다.
- 리팩터링 전후로 테스트가 GREEN 상태를 유지해야 한다.
- **새 동작 추가 금지.** 새 동작이 필요하면 다음 RED 사이클로 넘어가라.

### 반복

REFACTOR가 끝나면 다음 동작에 대한 RED로 돌아간다.

## 3. 삭제 강제 규칙

> **테스트 전에 프로덕션 코드를 먼저 작성했다면, 그 코드를 삭제하고 RED부터 재시작한다.**

- "이미 작성했으니 테스트만 붙이겠다"는 TDD가 아니다.
- **"참고용으로 남겨두기"도 금지.** 코드가 존재하면 테스트보다 코드를 먼저 읽게 된다.
- 삭제가 아깝다면: 그 코드는 테스트 없이 맞다고 확신할 수 없다.

---

## 4. 변명 차단표

| 변명 | 반론 |
|------|------|
| "너무 단순해서 테스트가 불필요하다" | 단순한 코드도 회귀한다. 단순하면 테스트 작성도 30초다. |
| "나중에 테스트를 추가하겠다" | 나중은 오지 않는다. 코드가 익숙해질수록 테스트 작성이 더 어려워진다. |
| "시간이 없다" | 버그를 디버깅하는 시간이 더 길다. TDD는 속도를 늦추지 않는다. |
| "삭제하면 작성한 코드가 낭비다" | 테스트 없는 코드는 동작을 보장할 수 없다. 낭비는 그 코드다. |
| "프로토타입이라 어차피 버린다" | 프로토타입은 프로덕션이 된다. 항상. |

---

## 5. 프로젝트별 규칙 우선 조항

- 이 파일은 **프로젝트 기본값(fallback)** 이다.
- `AGENTS.md`, `server/AGENTS.md`, `src/AGENTS.md`에 TDD 관련 지침이 명시되어 있으면 **그 지침이 우선**한다.
- 규칙 충돌 시 더 구체적인 파일(하위 디렉터리 AGENTS.md)이 이긴다.
